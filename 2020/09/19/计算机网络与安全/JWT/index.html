JWT

Posted on 2020-09-19 Edited on 2023-02-08 In 计算机网络与安全

JSON Web Token（缩写 JWT）是目前最流行的跨域认证解决方案

  • 认证：用户认证就是判断一个用户的身份是否合法的过程
  • 授权：控制用户访问资源

1 数据结构

  • Base64UrlEncode[header]**.Base64UrlEncode[payload].**Base64UrlEncode[signature]
  • decode 后结构：

1.1 头部（header)

是一个 JSON 对象

  • 声明类型，这里是 jwt
  • 声明加密的算法：
    • RS256。非对称算法，易于使用方获取和使用公钥
    • HS256。对称算法，必须注意确保密钥不被泄密

1.2 载荷（payload)

Payload 部分也是一个 JSON 对象，用来存放实际需要传递的数据

  • JWT 规定了 7 个官方字段，供选用。
    • iss (issuer)：签发人
    • exp (expiration time)：过期时间
    • sub (subject)：主题
    • aud (audience)：受众
    • nbf (Not Before)：生效时间
    • iat (Issued At)：签发时间
    • jti (JWT ID)：编号
  • 自定义信息
    • 通过 TokenEnhance 实现

1.3 签名（signature)

Signature 部分是对前两部分的签名，防止数据篡改。
使用 secret，对编码后的 header 和 payload 通过 header 中指定签名算法做数字签名，公式如下：
header中指定签名算法( ``**base64UrlEncode(header)+"."+base64UrlEncode(payload)** ,``**secret**``)

  • 算法
    • HS256
      • 先做摘要 再用 MAC 对称加密算法加密
    • RS256
      • 先做摘要 再用 RSA 非对称加密算法加密
